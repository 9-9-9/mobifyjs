Run the mobile site locally

If you want to see a more in-depth example of Mobify.js then what came with your initial scaffold, it's easy to download the code and make modifications locally.

  • Install Mobify (incase you haven't):

    $ sudo npm -g install mobify-client

  • Download a zip of the examples from Github:

    https://github.com/mobify/mobifyjs-demos/zipball/master

  • Unzip the folder, go into the demostore directory and run the preview server:

    $ cd demostore

    $ mobify preview

The Mobify.js tag is typically hard-coded to load the Mobify.js file to the location you specify. In order to load this file locally without changing the Mobify tag on the Demo Store, it uses a special tag that allows you to dynamically swap the Mobify.js file being loaded (it is the default tag in Mobify Cloud). But you'll need to authorize the swap first.

  • Go to preview.mobify.com and click "Authorize"

You should now see the mobile version of the website running off your local computer!

Play around with the code

With the preview server running on your local computer, you can edit the files and make adjustments to the Demo Store by refreshing the page. Here is an explaination of the most relevant files:

  • src/mobify.konf is where the DOM manipulation, content selection, and template rendering logic happens.

  • src/tmpl/*.tmpl is where the mobile templates exist. base.tmpl is included as the base for all page templates.

  • src/scss/* is where the mobile stylesheets exist. We used SCSS instead of regular old CSS. Check out the SCSS site to learn more.

If you want to understand more about how this stuff works, be sure to check out our documentation, and if you have any questions, head over to our community page.
